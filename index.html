ActionBar

Specifications

Design Spec: Abstract Spec

Overview

ActionBar holds up to two IconButtons and one PrimaryButton, and places itself at the bottom of its parent. It also can act as a container that is fixed at the bottom of its parent.

How to Use

ActionBar will auto size to its parent’s width, and adjust its height based on its children. It can be put into the container together with other contents, and stay at the bottom of that container. ActionBar’s default style accepts 0 to 2 IconButton props and exactly one PrimaryButton prop. Another way to use ActionBar is using it as a container, with no props but some children as contents.

...
return Roact.createElement(ActionBar, {
    button = {
        props = {
            onActivated = function() end,
            text = "Button",
            icon = Images["icons/common/notificationOff"],
        },
    },
    enableButtonAtStart = true,
    icons = {
        {
            props = {
                anchorPoint = Vector2.new(0.5, 0.5),
                position = UDim2.fromScale(0.5, 0.5),
                icon = Images["icons/common/notificationOff"],
                userInteractionEnabled = true,
                onActivated = function()
                    print("Text Button Clicked!")
                end,
                onStateChanged = function(oldState, newState)
                    print("state changed \n oldState:", oldState, " newState:", newState)
                end
            }
        }
    }
})

-- OR --

return Roact.createElement(ActionBar, {}, {
  Child = Roact.createElement("Frame")
})
...

Optional Props

  • button: table
    See Button for required and optional props.

  • enableButtonAtStart: boolean
    Indicate whether the button should be rendered at start of the action bar

  • icons: array

    • props: table
    • See IconButton for required and optional props.

  • marginOverride: table
    Overrides the inner margins of the action bar

  • horizontalAlignment: Enum.HorizontalAlignment = Enum.HorizontalAlignment.Center
    Overrides the horizontal alignment of the action bar contents. Default: center

BaseMenu

Design Specifications

Abstract link

Overview

A BaseMenu is a menu that contains a collection of cells.

How to Use

Menu = Roact.createElement(BaseMenu, {
    buttonProps = {
        {
            icon = Images["component_assets/circle_17"],
            text = "Option 1",
            onActivated = function()
                print("Option 1 pressed")
            end,
        },
        {
            icon = Images["component_assets/circle_17"],
            text = "Option 2",
            keyCodeLabel = Enum.KeyCode.Tab,
            selected = false,
            onActivated = function()
                print("Option 2 pressed")
            end,

            iconColorOverride = Color3.new(1, 0, 0),
            textColorOverride = Color3.new(1, 0, 0),
        },
        {
            icon = Images["component_assets/circle_17"],
            text = "Option 3",
            selected = true,
            onActivated = function()
                print("Option 3 pressed")
            end,
        },
    },

    width = UDim.new(0, 300),
    position = UDim2.fromScale(0.5, 0.5),
    anchorPoint = Vector2.new(0.5, 0.5),
    layoutOrder = 2,
})

Required Props

  • buttonProps : table
    the properties for each cell. It is an array that contains multiple tables of the following format:

    • text : string
      the label for the specific button

    • onActivated : function
      the function that fires when the button is clicked

    ### Optional

    • icon : table or string
      an rbxasset address recognized by Roblox Studio or an ImageSet Image.

    • keyCodeLabel : Enum.KeyCode
      a keycode to display a button hint for on the right side of the button

    • inputBindingKey: string
      a string localization key pased into the gamepad input binding for use in a gamepad bottom bar

    • selected : boolean
      a boolean value of if the cell is selected or not

    • iconColorOverride : Color3
      a Color3 value to override the Icon ImageColor with

    • textColorOverride : Color3
      a Color3 value to override the Text TextColor with

    • disabled : boolean
      if the cell is disabled or not

Optional Props

  • width : UDim = UDim.new(1, 0)
    the width of the menu

  • position : UDim2 = UDim2.new(0, 0, 0, 0)
    the position of the menu

  • anchorPoint : Vector2 = Vector2.new(0, 0)
    the anchor point of the menu

  • layoutOrder : number = 0
    the layout order of the menu

Specifications

Design Spec: Abstract Spec, Figma Spec for standard sizes

Overview

There are 4 basic buttons * Primary Contextual Button * Primary System Button * Secondary Button * Alert Button

Text and icon content type can be added any of the buttons * If icon is nil then it will be a text only button * If text is nil then it will be a icon only button * If text and icon are nil it will be a blank button * Else it will be a icon and text button

How to Use

Each of these buttons can be used with or without a standard size (Regular/Small/XSmall).

Without a standard size: These buttons will auto size to the parent so the best way to use a button is to wrap it with a Frame and have the frame control the size and positioning of the button.

With a standard size: Each standard size will apply the corresponding font style, padding, and button height for that standard size. These presets will override the size prop. The standard buttons can either fill their parent’s width by default, or have their width scale automatically to fit the text they contain. By default, Regular and Small buttons take the full size and XSmall buttons scale to fit content, but these defaults can be overridden with the fitContent prop. Regardless of this behavior, all standard buttons will never surpass a max width which is 640pts by default, but can be overridden with the maxWidth prop. If the text in any of these buttons is longer than its max width, or if the text in a Regular or Small button is wider than the parent container, the text will stay on one line but will be truncated at the end…

...
return Roact.createElement(PrimaryContextualButton, {
    size = UDim2.new(0, 144, 0, 48),
    layoutOrder = 1,
    text = "Button",
    isDisabled = disabled,
    isLoading = loading,
    onActivated = function()
        print "Primary Contextual Button Clicked!"
    end,
})
...

Required Props

  • onActivated: callback
    Callback for the activated event.

Optional Props

  • size: UDim2
    The size of the button.

  • standardSize: StandardButtonSize
    The standard button size the button should use: Regular, Small, or XSmall.

  • maxWidth: number
    The maximum width in pts beyond which the button should not extend. (Standard buttons only.)

  • fitContent: boolean
    Whether the button width should be based on the text length. (Standard buttons only. Default depends on standard size.)

  • anchorPoint: Vector2
    The size of the button.

  • position: UDim2
    The size of the button.

  • layoutOrder: number
    The size of the button.

  • icon: Image
    The icon of the button. A Image is either a ImageSetData or URL string.

  • text: string
    The text of the button.

  • inputIcon: Image
    The icon of the controller input (A/B/X/Y button) needed to activate the button

  • isDisabled: boolean = false
    Is the button disabled.

  • isLoading: boolean = false
    Is the button loading.

  • isHoverBackgroundEnabled: boolean = false
    Determine whether hover background is enabled

  • isDelayedInput: boolean = false
    Is the button going to delay input.

  • enableInputDelayed: boolean = false
    Used to start the input delay, set to true when you want to start the animation.

  • delayInputSeconds: number = 3
    Amount of seconds to delay input

  • userInteractionEnabled: boolean = true
    A Boolean value that determines whether user events are ignored and sink input

  • [React.Tag]: string
    Forward React tags to the underlying GuiButton instance.

ContextualMenu

Design Specifications

Abstract link

Overview

A DropdownMenu is a Select component that contains a list of values. Users can open the Menu and select a value from the Menu’s Cells.

The menu will be sized based on the size props. It contains a DropdownMenuList component to display the values. The DropdownMenuList will have the same width as the size props, and shows upon or under the DropdownMenuCell based on its absolute position relative to the screenSize props. If the screenSize has a width under 640pt, the DropdownMenuList will stick to the bottom of the screenSize with a 24pt padding. The dismiss modal background will be sized to the full screen size, independant of parent size.

How to Use

DropdownMenu = Roact.createElement(DropdownMenu, {
  placeholder = "Make a Choice",
  onChange = function(value) print(value) end,
  size = UDim2.fromOffset(300,48),
  screenSize = UDim2.fromOffset(640,800),
  errorState = false,
  showDropShadow = false,
  fixedListHeight = nil,
  cellDatas = {
    {
      text = "Item 1",
    },
    {
      text = "Item 2",
    },
    {
      text = "Item 3",
    },
    {
      text = "Item 4",
    },
    {
      text = "Item 5",
    },
    {
      text = "Item 6",
    },
    {
      text = "Item 7",
    },
    {
      text = "Item 8",
    },
  }
})

Required Props

  • cellDatas : table
    the properties for each cell. It is an array that contains multiple tables of the following format:

    • text : string
      the label for the specific button

    ### Optional

    • icon : table or string
      an rbxasset address recognized by Roblox Studio or an ImageSet Image.

    • keyCodeLabel : Enum.KeyCode
      a keycode to display a button hint for on the right side of the button

    • inputBindingKey: string
      a string localization key pased into the gamepad input binding for use in a gamepad bottom bar

    • iconColorOverride : Color3
      a Color3 value to override the Icon ImageColor with

    • textColorOverride : Color3
      a Color3 value to override the Text TextColor with

    • disabled : boolean
      if the cell is disabled or not

  • size : UDim2
    the size of the DropdownMenu, actually represents the DropdownMenuCell’s size and DropdownMenuList’s width.

  • screenSize : Vector2
    the total size of the screen, used for the dismiss background and the DropdownMenuList’s position on the compact width.

  • onChange : function
    a function to be called when a Cell is clicked , passing the value as a parameter. This will be called even if the value is the same with the previous one.

Optional

  • isDisabled : boolean
    if the DropdownMenu is disabled.

  • errorState : boolean
    if the DropdownMenu is in the error state.

  • showDropShadow : boolean
    if the DropdownMenu’s DropdownMenuList’s frame has shadow effect.

  • fixedListHeight : number
    if the DropdownMenu’s DropdownMenuList’s height with fixed number.
